## v0.4.0 — rules accept bare markdown

Rules no longer require a YAML frontmatter block. A `.md` file with just markdown body — no `---`, no metadata — is now a valid rule. Skills, agents, instructions, and commands keep the strict requirement.

### Why

Claude itself doesn't consume rule frontmatter — the metadata `agtk` was demanding (`description`, `name`) was tooling overhead, not something the agent needed. For Claude-only consumers that meant inventing a description for every rule just to satisfy the parser. The friction wasn't pulling weight.

The forward-looking concern was Cursor: Cursor's `.mdc` rules need `description`/`globs`/`alwaysApply`. The plan there is for the Cursor adapter to **synthesize** those fields from defaults at render time rather than push the requirement onto every rule author up front.

Skills and agents stay strict because the metadata they carry (description used for routing/discovery, tool allowlists, argument hints, platform allowlists) is real adapter input, not just bookkeeping.

### What changed

- **`internal/definitions/parser.go`** — when a rule file has no leading `---`, the whole file is treated as the body. `Common.Description` may be empty for rules; `Common.Name` falls back to the filename stem as it always has.
- **`internal/adapters/claude/files.go`** — `description` is now `omitempty` in the rendered rule frontmatter, so bare-markdown rules don't produce ugly `description: ""` lines.

### Examples

All three of these are now valid rule files:

```markdown
# my-rule

Just a body. No frontmatter at all.
```

```markdown
---
name: my-rule
paths:
  - "src/**"
---

Frontmatter present, but description omitted.
```

```markdown
---
name: my-rule
description: Path-scoped style rule.
paths:
  - "src/**/*.tsx"
---

Frontmatter with full metadata — unchanged.
```

### Compatibility

- No stack-manifest schema changes. `.agentic-toolkit.yaml` and lockfiles parse identically to v0.3.x.
- Files that previously failed to parse (rule `.md` with no frontmatter, or rule frontmatter without `description:`) now succeed. No previously-valid input becomes invalid.
- Rendered rule output: `description:` is dropped from the Claude-rendered frontmatter when empty; rules that always set a description render byte-identically to v0.3.x.
