## v0.7.0 — `--source` flag, skill frontmatter, and a refreshed definition catalog

A feature release on top of v0.6.0: a new way to apply a stack from a toolkit tree elsewhere on disk, skill frontmatter passthrough for Claude, and a large refresh of the shipped definitions (a new multi-agent review skill, plus Serena/RTK integrations and language-specific stacks).

### New: `--source <dir>` `[--stack <name>]`

A global flag to apply a stack from a toolkit source tree that lives **somewhere else on disk**, exactly as if `agtk` were run inside it:

```bash
# render this toolkit's `rust` stack into the current project,
# sourcing skills/agents/hooks from a sibling checkout:
agtk --source ../agentic-toolkit --stack rust sync
```

- Bare-name definitions resolve against `<dir>/definitions/`; the entry manifest is `<dir>/.agentic-toolkit.yaml` by default, or `<dir>/stacks/<name>.yaml` when `--stack` names one.
- Rendered output **and** the lockfile land in the working directory, so the shared/read-only source tree is never written to.
- Mutually exclusive with `--config`. The change is CLI-layer only — the resolver is unchanged.

Use it for local development of a toolkit before pushing, or to apply a shared checkout without going through a git URL. Documented in `docs/CONSUMER-GUIDE.md`.

### New: skill frontmatter — `argument_hint` and `disable_model_invocation`

Skill definitions can now carry Claude-specific frontmatter under `extensions.claude`, rendered straight into the generated `SKILL.md`:

```yaml
extensions:
  claude:
    argument_hint: "What is the goal?"
    disable_model_invocation: true
```

### Definitions: new multi-agent review skill

- **`pre-pr-review`** (replaces `code-panel-review`) — a multi-agent pre-PR review that picks a stack-specific reviewer panel from a data-driven `agents.json` (`detect` + `agents` maps), fans specialists out in parallel, and triages findings RED/AMBER/GREEN. Ships panels for `kotlin-spring`, `react`, `rust`, and `go`.
- New skills: `agents-md-creator`, `continuation-session`, `wrap-session`. `challenge` reworked (one-question-at-a-time interview restored, plus CONTEXT/ADR templates).
- New agents: `code-explorer`, `wrap-session-reviewer`.

### Definitions: Serena, RTK, and language stacks

- **Serena**: a `serena` MCP server definition plus `serena-*` lifecycle hooks (install, activate, remind, cleanup).
- **RTK**: `rtk-claude-session-start` (install) and `rtk-claude-pre-tool-use` (ensure used) hooks.
- New shareable stacks: `stacks/rust.yaml`, `stacks/go.yaml`, `stacks/serena.yaml`, `stacks/rtk.yaml`. `stacks/default.yaml` now bundles `pre-pr-review`, `agents-md-creator`, `continuation-session`, `wrap-session`, the `code-explorer` agent, and a `git` instruction.

### Compatibility

- **No schema changes** — manifests and lockfiles parse identically to v0.6.x. `extensions.claude.argument_hint` / `disable_model_invocation` are additive optional fields.
- **No breaking CLI changes** — `--source`/`--stack` are additive; all existing invocations behave as before.
- **Definition removal:** the `code-panel-review` skill was removed in favor of `pre-pr-review`. `stacks/default.yaml` was updated accordingly, but any consumer that referenced `code-panel-review` directly (by bare name or URL) must switch to `pre-pr-review`.
