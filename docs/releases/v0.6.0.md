## v0.6.0 — Plannotator stack + AGENTS.md fix for `--config`

Two changes on top of v0.5.0: a new shareable stack that wires [plannotator](https://plannotator.ai) into Claude Code's plan-approval flow, and a fix to the AGENTS.md seeding logic for the bare-repo + worktree workflow introduced in v0.5.0.

### New: plannotator stack

A new shareable stack at `stacks/plannotator.yaml` that turns on plan-review annotation in Claude Code with one extends line:

```yaml
extends:
  - github.com/pedromvgomes/agentic-toolkit/stacks/plannotator.yaml@main
```

Renders two hooks:

- `plannotator-claude-session-start` — installs `plannotator` from `https://plannotator.ai/install.sh` if it isn't already on PATH for the session.
- `plannotator-claude-exit-plan-mode` — runs `plannotator` against the plan on `ExitPlanMode` so it gets annotated before approval.

### Fix: AGENTS.md is now sourced from the stack directory under `--config`

In v0.5.0, when you pointed `--config` at a manifest in a worktree (e.g. `agtk --config main/.agentic-toolkit.yaml sync` from a bare-repo root), the CLAUDE.md seeder only checked the apply directory for `AGENTS.md`. A worktree-local `main/AGENTS.md` next to the manifest was silently ignored.

v0.6.0 looks for `AGENTS.md` at the **stack directory** first (next to the manifest), falls back to the apply directory, and emits the seeded `@`-import as the path **relative to CLAUDE.md's location** so the agent resolves it correctly at runtime:

- No `--config` (single-directory case): `@AGENTS.md` — unchanged.
- Bare-repo + worktree: `@main/AGENTS.md` — now found and imported.

If `AGENTS.md` exists at both locations, the stack directory wins (the manifest is the project definition).

### Internal: `configDir` → `stackDir`

The internal CLI helper that points at the manifest's directory was renamed from `configDir` to `stackDir`, since it now roots four things (manifest, lockfile, local `./...` refs, and AGENTS.md) and "config" had grown too narrow. A new `StackDir` field on `claude.Options` plumbs it through to the renderer; empty defaults to `ProjectRoot`, so callers that don't set it see no behavior change.

### Compatibility

- No schema changes — manifests and lockfiles parse identically to v0.5.x.
- No CLI surface changes.
- The AGENTS.md lookup is now strictly *more* permissive: any setup that worked on v0.5.0 still works on v0.6.0; setups that had a `stackDir/AGENTS.md` that was previously ignored now get it imported.
