## v0.1.0 — first public release

The first tagged release of `agtk`, the agentic-toolkit CLI. This release packages the full slice plan that landed in PRs #5–#13 and produces the cross-platform archives the auto-updater downloads on subsequent versions.

### What's in the box

A consumer workflow for declaring, locking, fetching, and rendering a stack of agentic-coding definitions (skills, agents, commands, rules, instructions, hooks, MCP servers, and arbitrary settings.json fragments) onto a host Claude Code installation.

### Definition catalog

- Eight typed categories with frontmatter validation: `skill`, `agent`, `command`, `rule`, `instruction`, `hook`, `mcp`, `setting`.
- Bundle-shaped categories (`skill`, `agent`) keep their companion files alongside the entry point; file-shaped categories live in flat directories.
- External preset refs use the grammar `<plural>::<repo>.git/<path>[@<ref>]` — the `.git/` boundary is the explicit split between the repository URL and the in-repo path.
- See `definitions/SCHEMA.md` for the full per-category schema.

### Resolver

- `agtk lock` resolves every preset entry against the network and writes a deterministic lockfile (`.agentic-toolkit.lock.yaml`).
- `agtk fetch` hydrates the on-disk source cache from the lockfile without ever touching the network for ref resolution.
- `agtk plan` prints the resolved Plan in frozen-lockfile mode (`--json` for scripting; `--quiet` to suppress diagnostics).
- Override losers and implicit-external sources surface as structured diagnostics rather than silent surprises.

### Renderer

- `agtk render [--scope user|project] [--dry-run] [--force]` emits Claude Code's expected layout under `<scope>/.claude/`.
- Three ownership models coexist:
  - **Whole-owned files** (skills/agents/commands/rules) tracked in `.agtk-manifest.json`; existing untracked files refuse to render unless `--force`.
  - **Managed-region** `CLAUDE.md` with `<!-- BEGIN/END AGTK MANAGED -->` markers; user content outside the markers is preserved. Project scope seeds new files from `AGENTS.md` via `@AGENTS.md` import.
  - **Mixed-ownership** `settings.json` with `_meta.agtk.managed` listing the top-level keys agtk owns.

### Observability + CI guardrails

- `agtk status` reports drift across three buckets (config vs lockfile, lockfile vs cache, rendered state vs plan); exits non-zero on any drift.
- `agtk lock --frozen` re-resolves but refuses to write — fails the build when the lockfile would change.
- `--json` (versioned, `version: 1`) on `plan`, `lock`, and `status` for stable scripting output.

### Auto-update

- Background update check runs on every CLI invocation when stdout is a terminal, the binary has a real version tag, the user hasn't disabled it, and the throttle interval has elapsed.
- A one-line stderr notice surfaces a newer release without delaying the main command.
- `agtk update` (interactive), `agtk update --yes`, `agtk update --check` (exit 10 when newer is available).
- Self-replace verifies the archive against the release's `checksums.txt` before applying.

User config: `${XDG_CONFIG_HOME:-~/.config}/agentic-toolkit/config.yaml` (`auto_update.enabled`, `check_interval`).
Throttle state: `${XDG_STATE_HOME:-~/.local/state}/agentic-toolkit/state.yaml`.

### Install

```sh
go install github.com/pedromvgomes/agentic-toolkit/cmd/agtk@v0.1.0
```

Or download the pre-built archive for your platform from the assets below and place `agtk` on `$PATH`.

### Known caveats

- The Claude renderer's frontmatter shapes for skills/agents/commands target Claude Code's current vocabulary; future Claude Code schema changes may require adjustments.
- `rule` definitions don't yet expose a `paths:` frontmatter field for path-scoped auto-attach; tracked for a follow-up.
- `agtk update` is unauthenticated against the GitHub API (60 req/hr per IP) — fine for personal use.
